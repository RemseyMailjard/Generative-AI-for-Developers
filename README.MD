# GitHub Copilot Instructions & Skills

## Overview
This guide covers GitHub Copilot usage patterns, best practices, and examples for leveraging AI-assisted coding.

## Instructions

### 1. Code Completion
Ask Copilot to complete functions:
```javascript
// Start typing and Copilot suggests completions
function calculateSum(arr) {
    // Copilot completes the implementation
}
```

### 2. Generate Tests
Request unit tests:
```python
# Write test cases for your functions
def test_user_authentication():
        # Copilot generates test implementations
```

### 3. Documentation
Create JSDoc/docstrings:
```typescript
/**
 * Copilot helps generate comprehensive documentation
 */
export function processData(input: string): void {}
```

## Core Skills

- **Code Generation**: Copilot suggests full functions and blocks
- **Language Translation**: Convert code between programming languages
- **Refactoring**: Improve code structure and readability
- **Debugging**: Identify issues and suggest fixes
- **Test Creation**: Generate unit and integration tests
- **Documentation**: Write comments and docstrings
- **SQL/Query Writing**: Create database queries

## Best Practices

1. Write clear comments describing what you need
2. Provide context and examples
3. Use Copilot suggestions as a starting point
4. Always review generated code
5. Test thoroughly before deploying

## Tips for Better Results

- Use specific, descriptive variable names
- Include type hints in your code
- Reference existing patterns in your codebase
- Break complex problems into smaller tasks
